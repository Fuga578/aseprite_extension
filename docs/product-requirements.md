# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**AseCLI**(アセCLI) - Aseprite画面内でCLIを実行するコンソール拡張

> 名称は仮称です。承認時に変更可能です(itch.io検索性を考えると "CLI Console for Aseprite" 等の併記も有効)。

### プロダクトコンセプト
- **Aseprite内蔵CLIコンソール**: Asepriteを離れず、画面内のパネルから `aseprite -b ...` バッチコマンドを実行できる
- **擬似ターミナルの操作感**: コマンド入力欄・実行ボタン・出力ログという、ターミナルに近いシンプルなUI
- **バッチ自動化の入口**: スプライトシート生成・一括フォーマット変換など、GUIに代替手段が無いCLI操作を手元で完結させる

### プロダクトビジョン
ドット絵ツール Aseprite には「GUIからは実行できないが、CLI(コマンドラインインターフェース)からなら実行できる」操作が数多く存在する。AseCLI は、それらのCLI操作を Aseprite の画面内から直接実行できるようにする拡張機能である。ユーザーはターミナルアプリへ画面を切り替えることなく、制作のコンテキストを保ったままバッチ処理を実行できる。まずは上級者向けの「生コマンド入力ターミナル」として最小構成でリリースし、将来的にはCLIフラグを知らなくても使えるGUIへ発展させる。

### 目的
- Aseprite ↔ ターミナル間の画面切り替えコストをなくす
- 散らかりがちなバッチスクリプト(.bat/.sh)の管理を不要にする
- GUIに存在しないCLIバッチ操作を、Asepriteユーザーが日常的に使えるようにする
- フリーミアムモデルで小規模ながら継続的な収益を得る

## ターゲットユーザー

### プライマリーペルソナ: 佐藤健太(28歳、インディーゲーム開発者)
- 個人または2〜3人のチームでドット絵のゲームを開発している
- Aseprite を主力ツールとして毎日数時間使用(Aseprite v1.3 系を利用)
- スプライトシートのエクスポートや解像度別の書き出しを頻繁に行う
- **現在の課題**: 書き出しのたびにターミナルを開き、`aseprite -b ...` の長いコマンドを手打ち or バックスクロールして再利用している。Aseprite と行き来するたびに集中が切れる
- **期待する解決策**: Aseprite の画面内でコマンドを打って実行し、結果(成功/失敗・出力ファイル)をその場で確認したい
- **1日の典型的なワークフロー**: スプライト編集 → 保存 → スプライトシート/各種フォーマットへ書き出し(ここでCLIを使用) → ゲームエンジンで確認 → 修正、を1日に何度も繰り返す

### セカンダリーペルソナ: 山田美咲(32歳、ドット絵アセット制作者)
- itch.io 等でドット絵アセットパックを制作・販売している
- 同じ書き出し作業(PNG/GIF化、サイズ違いの量産)を大量に繰り返す
- CLIは「コピペで使う」程度の知識。フラグを一から組むのは苦手
- **現在の課題**: 繰り返し作業が多く、手作業のミス(書き出し忘れ・設定間違い)が発生する
- **期待する解決策**: よく使うコマンドを保存しておき、ワンクリックで再実行したい(→ Pro版のプリセット機能が刺さる層)

### 想定しないユーザー
- CLIの概念をまったく知らない初心者(MVPの生コマンド入力UIでは離脱する。将来のフォーム型UIで対応)
- Aseprite v1.3.7 未満を使い続けるユーザー(技術的に動作しないため)

## 機能要件

### コア機能(MVP)

#### 機能1: コマンド入力・実行パネル

**ユーザーストーリー**:
インディーゲーム開発者として、Asepriteを離れずにバッチ処理を実行するために、コマンドを入力して実行できるパネルが欲しい。

**受け入れ条件**:
- [ ] Asepriteのメニュー(またはパレット)からパネルを開ける
- [ ] 1行のコマンド入力欄があり、`aseprite -b sprite.ase --sheet sheet.png` のようなコマンドを入力できる
- [ ] 入力欄の先頭トークン `aseprite` は、設定されたAseprite実行ファイルパスに自動置換される
- [ ] 「Run」ボタン、または入力欄でのEnterキーで実行できる
- [ ] 実行中はボタンが無効化され、二重実行を防ぐ
- [ ] 実行は `os.execute()` を用い、Asepriteとは別プロセスでバッチモード実行される

**優先度**: P0(必須)

#### 機能2: 実行結果の表示

**ユーザーストーリー**:
ユーザーとして、コマンドが成功したか失敗したかをその場で把握するために、実行結果(出力・終了コード)の表示が欲しい。

**受け入れ条件**:
- [ ] コマンドの標準出力・標準エラー出力を一時ファイル経由で取得し、パネル内のログ領域に表示する
- [ ] 終了コード(exit code)と、成功/失敗が視覚的に分かる(例: ✓ / ✗ と色分け)
- [ ] 実行したコマンド文字列もログに記録され、何を実行したか後から分かる
- [ ] ログは複数回の実行分を時系列で蓄積表示する(セッション中)
- [ ] 出力が長い場合もスクロールまたは省略表示で破綻しない

**優先度**: P0(必須)

#### 機能3: Aseprite実行ファイルパスの設定

**ユーザーストーリー**:
ユーザーとして、自分の環境でコマンドを動かすために、Aseprite実行ファイルの場所を設定したい。

**受け入れ条件**:
- [ ] 設定画面でAseprite実行ファイル(`aseprite.exe` 等)のパスを指定できる
- [ ] 可能な場合は一般的なインストール先を初期値として自動推定する
- [ ] パスが未設定/不正な場合、実行前に分かりやすいエラーメッセージを表示する
- [ ] 設定値はAseprite再起動後も保持される(永続化)

**優先度**: P0(必須)

#### 機能4: 作業ディレクトリの管理

**ユーザーストーリー**:
ユーザーとして、相対パスでファイルを指定できるように、コマンドの実行ディレクトリを管理したい。

**受け入れ条件**:
- [ ] コマンドの実行時の作業ディレクトリ(カレントディレクトリ)が定義されている
- [ ] 既定値は「現在開いているスプライトのあるフォルダ」とし、未保存スプライトの場合は設定された既定ディレクトリを使う
- [ ] 作業ディレクトリを手動で変更でき、現在の作業ディレクトリがパネル上に表示される

**優先度**: P0(必須)

#### 機能5: コマンド履歴

**ユーザーストーリー**:
ユーザーとして、同じコマンドを再実行する手間を省くために、過去に実行したコマンドを呼び出したい。

**受け入れ条件**:
- [ ] 入力欄で ↑ / ↓ キー、またはボタン操作で直近の実行コマンドを呼び出せる
- [ ] 履歴はセッション内で最低20件保持される
- [ ] (可能なら)履歴はAseprite再起動後も保持される

**優先度**: P1(重要)

#### 機能6: 未保存スプライトに対する実行時の警告

**ユーザーストーリー**:
ユーザーとして、古いファイルを処理する事故を防ぐために、未保存の変更がある状態でCLIを実行しようとしたら警告してほしい。

**受け入れ条件**:
- [ ] CLIはディスク上のファイルを処理し、編集中(メモリ上)の変更は反映されない旨を、初回または該当時に明示する
- [ ] 現在開いているスプライトに未保存の変更があり、かつコマンドがそのファイルを参照する可能性がある場合に警告を表示する

**優先度**: P1(重要)

#### 機能7【Pro】: コマンドプリセットの保存

**ユーザーストーリー**:
アセット制作者として、繰り返す書き出し作業を効率化するために、よく使うコマンドを保存してワンクリックで再実行したい。

**受け入れ条件**:
- [ ] 任意のコマンドに名前を付けて保存できる
- [ ] 保存済みプリセットを一覧から選んで入力欄に呼び出せる
- [ ] プリセットはAseprite再起動後も保持される
- [ ] この機能はPro版でのみ有効。無料版ではアップグレード導線が表示される

**優先度**: P1(重要・Pro版機能)

### CLIインターフェース(パネル内で入力するコマンド例)

```bash
# スプライトシートの書き出し
aseprite -b player.ase --sheet player-sheet.png --data player.json

# フォーマット一括変換(全フレームをPNG連番に)
aseprite -b animation.ase --save-as frame{frame}.png

# 2倍に拡大して書き出し
aseprite -b icon.ase --scale 2 --save-as icon@2x.png

# 指定レイヤーのみ書き出し
aseprite -b character.ase --layer "Body" --save-as body.png
```

> ユーザーは先頭を `aseprite` と書く。拡張が実行時に設定済みの実行ファイルパスへ置換する。
> `-b`(バッチモード)が必要なケースが多いことは、UI上のヒントやサンプルで案内する。

### 将来的な機能(Post-MVP)

#### 機能8【Pro】: プレースホルダ付きコマンドテンプレート
`aseprite -b {input} --scale {n} --save-as {output}` のように変数を埋め込んだテンプレートを保存し、実行時に値を入力できる。
**優先度**: P2(できれば)

#### 機能9【Pro】: 複数コマンドのバッチ実行
複数のコマンドを順に連続実行する。
**優先度**: P2(できれば)

#### 機能10: フォーム型ランチャー
CLIフラグを知らなくても、項目を選ぶだけで主要なバッチ操作(書き出し・変換等)を実行できるGUI。CLI初心者層へ対象を拡大する。
**優先度**: P2(できれば)

#### 機能11: macOS / Linux 最適化
クオート/エスケープ処理のOS差異に対応し、Windows以外でも安定動作させる。
**優先度**: P2(できれば)

## 非機能要件

### パフォーマンス
- パネルの表示: 操作から500ms以内に開く
- 拡張側のコマンド送出オーバーヘッド(入力確定〜別プロセス起動まで): 200ms以内(実際のバッチ処理時間はAseprite本体に依存するため対象外)
- 出力ログ表示: 10,000行までスクロール操作が体感1秒以内で破綻しない
- コマンド実行は同期的(`os.execute` がプロセス終了まで待機)であり、実行中はAsepriteのUIがブロックされる。`aseprite -b` の一般的なバッチ処理は通常1秒未満で完了するため許容範囲とする。長時間処理が想定されるコマンドの非同期(切り離し)実行はPost-MVPで検討する
- 実行中はパネル上に処理中であることを示す表示を出し、ユーザーがフリーズと誤認しないようにする

### ユーザビリティ
- 新規ユーザーがインストール後5分以内に最初のコマンドを実行できる
- パネル内にコマンド例/ヒントを常設し、ドキュメントを開かずに基本操作が分かる
- エラーメッセージは原因と対処(例: 「実行ファイルパスが未設定です。設定画面で指定してください」)を日本語/英語で明示する
- セキュリティ許可ダイアログについて、初回オンボーディングで「なぜ表示されるか」「完全に信頼を選んでよいか」を説明する

### 信頼性
- コマンド実行の成否(終了コード)を取りこぼさず表示する
- 拡張側の処理でAseprite本体をクラッシュさせない(実行は別プロセスのため本体に影響しない設計)
- 一時ファイル(出力リダイレクト用)は実行後に確実に削除またはクリーンアップする
- 不正/空のコマンド入力時は実行せず、安全にエラー表示する

### セキュリティ
- 動作要件: **Aseprite v1.3.7 以降**(`os.execute` の戻り値が正しく取得できる最初のバージョン)
- ユーザーが「Run」を押した時以外、コマンドを実行しない(自動実行・バックグラウンド実行をしない)
- Asepriteのセキュリティ許可モデル(`os.execute` 実行時の許可ダイアログ)を尊重し、回避しない
- ユーザーの同意なくネットワーク通信・テレメトリ送信を行わない
- 入力されたコマンドはユーザー自身の責任で実行される旨を明示する(ライセンス購入確認等の外部通信を行う場合は別途明記)

### 互換性 / スケーラビリティ
- 主対象OS: Windows(`os.execute` が `cmd` 経由で動作)。macOS / Linux は P2 で対応
- Aseprite v1.3.7 以降のすべてのv1.3系で動作すること(将来のメジャー更新時はAPI差分を確認)
- 配布形式: `.aseprite-extension`(`package.json` を含むZIP)。itch.io 経由で配布
- フリーミアム: 無料版とPro版を提供。Pro機能はライセンス判定で有効化する(判定方式の詳細はアーキテクチャ設計で決定)

## スコープ外

明示的にスコープ外とする項目:
- **編集中(メモリ上)のスプライトを直接操作する機能**: CLIはディスク上のファイルを処理する。ライブ編集中ドキュメントの操作は `app.command` の領域であり対象外
- **内部アプリコマンド(`app.command.*`)の実行**: 標準の `Ctrl+Space`「Run Command」窓と重複するため対象外
- **本物のターミナルエミュレータ**: 対話的シェル(プロンプト、curses等)は再現しない。あくまで単発コマンドの実行と結果表示に限定
- **Aseprite以外の任意の外部コマンドの汎用実行**: 本拡張は「AsepriteのCLI」を実行する用途に絞る(汎用シェルランチャーにはしない)
- **フォーム型GUIランチャー**: MVPには含めない(機能10としてPost-MVPで検討)
- **クラウド同期・チーム共有機能**

## プロダクトビジョン

ドット絵ツール **Aseprite** の画面内から、AsepriteのCLI(コマンドラインインターフェース)を
実行できるようにする拡張機能。Asepriteには「GUIからは実行できないがCLIなら可能」な操作が
多数あるため、それらをAsepriteを離れずに使えるようにする。

**一言で言うと**: Aseprite内蔵のCLIコンソール

---

## 解決したい課題

1. **CLIバッチ操作がGUIから使えない**
   - スプライトシート生成、一括フォーマット変換、リサイズ等は `aseprite -b ...` でしか実行できない
   - そのたびにターミナルを開き、長いコマンドを手打ちする必要がある

2. **画面の切り替えコスト**
   - Asepriteで作業 → ターミナルに切り替え → コマンド実行 → Asepriteに戻る
   - 集中が途切れる

3. **バッチスクリプトの管理が面倒**
   - よく使うコマンドを.bat/.shファイルに保存しているが、散らかる・どこにあるか分からなくなる

---

## ターゲットユーザー(概要)

- **インディーゲーム開発者 / 技術寄りのドット絵師**
  - スプライトシートのエクスポートや一括変換を日常的に行う
  - CLIの概念は理解しているが、Asepriteを離れたくない
- **ドット絵アセット制作・販売者**
  - 同じエクスポート作業を大量に繰り返す

## 実現性の調査結果(2026-05 時点)

### できること
- 外部コマンド実行: Lua `os.execute()` で可能。**Aseprite v1.3.7 以降が必須**
  (v1.3.6 以前は戻り値が nil になるバグあり、v1.3.7 で修正済み)
- 拡張UI: `Dialog` API でパネル/ダイアログを作成可能
- `.aseprite-extension` として itch.io 等で配布・販売可能

### 制約・注意点
- **言語は Lua のみ**(Python ではない)。Aseprite拡張は Lua 5.4 + Aseprite Scripting API
- `os.execute` 実行時に Aseprite のセキュリティ許可ダイアログが出る(「完全に信頼」で以降は抑制)
- `io.popen` は不安定。出力(stdout)は一時ファイルにリダイレクトして読み戻す方式が確実
- AsepriteのCLIは常に**別プロセス**を起動し、**ディスク上のファイル**を処理する
  → 編集中(メモリ上)のスプライトには直接効かない。CLIは「ファイルの一括処理」向き
- `Dialog` API に複数行テキスト欄/本物のターミナルウィジェットは無い
  → 入力欄＋実行ボタン＋ログ表示の「擬似ターミナル」になる(`canvas` で自前描画も可)

### 「CLI」と「内部コマンド」の区別(重要)
- **CLIバッチ操作** (`aseprite -b ...`): GUIに代替が無い → 本拡張の中心的価値
- **内部アプリコマンド** (`app.command.*`): 標準の `Ctrl+Space`「Run Command」窓と重複 → 今回は対象外

## 競合状況

- **市場**: itch.io に Aseprite 関連ツール約319件。有料拡張も一般的(相場 $3〜$10)
- **直接競合**: 「CLIランナー/ターミナルパネル」としてパッケージ化された拡張は見当たらない(ニッチだが空きがある)
- **間接競合(標準機能)**: `Ctrl+Space`「Run Command」窓、`aseprite --shell`(対話コンソール)
- コミュニティではユーザーが各自スクリプトを自作している → 需要は存在するが層は薄い

---

## 方針決定(壁打ち結果)

| 論点 | 決定 |
|------|------|
| 対象コマンド | **CLIバッチ操作中心**(`aseprite -b ...`) |
| MVPのUI形式 | **生コマンド入力ターミナル**(入力欄＋実行＋ログ) |
| 収益化 | **フリーミアム**(基本無料＋有料Pro版) |

## 主要機能の候補

### P0 (MVPで必須)
1. コマンド入力・実行パネル(`aseprite XXX` を入力 → 実行)
2. 実行結果(標準出力/エラー出力/終了コード)の表示
3. Aseprite実行ファイルのパス設定
4. 作業ディレクトリの指定

### P1 (重要)
5. コマンド履歴(↑↓で呼び出し)
6. 未保存スプライトに対する実行時の警告
7. 【Pro】よく使うコマンドのプリセット保存

### P2 (将来)
8. 【Pro】プレースホルダ付きコマンドテンプレート
9. 【Pro】複数コマンドのバッチ実行
10. フォーム型ランチャー(CLIフラグを知らなくても使えるGUI)
11. macOS / Linux 最適化

---

## 収益化の考え方(フリーミアム)

- **無料版**: 単一コマンドの入力・実行・出力表示。これだけでも実用になる
- **Pro版(有料)**: プリセット保存、テンプレート、バッチ実行、ログのエクスポート等
  - 価格帯: $3〜$5 を想定(Aseprite拡張の相場)
  - 収益規模は小さい見込み。まず無料版でユーザーとレビューを獲得する戦略
